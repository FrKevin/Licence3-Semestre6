unsigned char st_dns_packet::authoritative_answer

Only for answer. Authoritative Answer - this bit is valid in responses, and specifies that the responding name server is an authority for the domain name in question section.

Note that the contents of the answer section may have multiple owner names because of aliases. The AA bit corresponds to the name which matches the query name, or the first owner name in the answer section.

unsigned char st_dns_packet::id[2]

A 16 bit identifier assigned by the program that generates any kind of query. This identifier is copied the corresponding reply and can be used by the requester to match up replies to outstanding queries.

unsigned char st_dns_packet::is_answer

type_of_request st_dns_packet::op_code

response_code st_dns_packet::r_code

Only for answer. Response code - this 4 bit field is set as part of responses.

unsigned char st_dns_packet::recursion_available

Only for answer. Recursion Available - this be is set or cleared in a response, and denotes whether recursive query support is available in the name server.

unsigned char st_dns_packet::recursion_desired

Recursion Desired - this bit may be set in a query and is copied into the response. If RD is set, it directs the name server to pursue the query recursively. Recursive query support is optional.

unsigned char st_dns_packet::reserved[3]

Reserved for future use. Must be zero in all queries and responses.
